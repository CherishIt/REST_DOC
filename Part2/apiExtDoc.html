About

This is the REST Extended API Document of ECS Library.


The extended APIs are designed for loaning books.

Design Decisions:
  1. Four new APIs are added: /loans /loans/<loan_id> /fines /fines/<fine_id>.
  2. PATCH is used to update while PUT WON'T be supported because only part of information need update.
  3. DELETE WON'T be supported as all loans and fines need to be recorded. Instead, a property status is used to denote the current status(e.g. status="Returned" denotes the end of a loan).
Loan Properties
Property Type Description
url string Url of the loan.
book string(valid url of book) Url of the loaned book.
date_loaned date Date when the book was loaned.
date_due date Date by when the book should be returned.
status string("Loaned"/"Overdue"/"Returned") Current status of the loan.
owner string Name of user who own the loan
Fine Properties
Property Type Description
url string Url of the fine.
loan string(valid url of loan) Url of the corresponding loan.
date_started date Date when the fine was started.
amount integer Fine amount(unit: p).
status string("Open"/"Closed") Current status of the fine.
owner string Name of user who owe the fine

Loans

ECS library offers /loans resource for providing the loans list or creating a new loan.

Once a new loan is created, /loans/<loan_id> can be used to get, update the loan.

/loans

Supported HTTP Methods: GET POST

GET

Description:

Using GET on this api to get a list of all loans.

Request:

Simply GET /loans

Sample Request
GET http://waisvm-rfp1.ecs.soton.ac.uk/loans
Response:

Returns a list of all loans and HATEOAS links.

Sample Response
<?xml version="1.0" encoding="utf-8"?>
<root>
    <loans>
        <list-item>
            <url>http://waisvm-rfp1.ecs.soton.ac.uk/loans/1</url>
            <book>http://waisvm-rfp1.ecs.soton.ac.uk/books/1</book>
            <date_loaned>2015-11-1</date_loaned>
            <date_due>2015-11-15</date_due>
            <status>Loaned</status>
            <owner>cz4n14</owner>
        </list-item>
        <list-item>
            <url>http://waisvm-rfp1.ecs.soton.ac.uk/loans/2</url>
            <book>http://waisvm-rfp1.ecs.soton.ac.uk/books/2</book>
            <date_loaned>2015-10-30</date_loaned>
            <date_due>2015-11-13</date_due>
            <status>Overdue</status>
            <owner>cz4n14</owner>
        </list-item>
    </loans>
    <links>
        <list-item>
            <href>http://waisvm-rfp1.ecs.soton.ac.uk/loans</href>
            <rel>self</rel>
            <method>GET</method>
        </list-item>
        <list-item>
            <href>http://waisvm-rfp1.ecs.soton.ac.uk/loans</href>
            <rel>create</rel>
            <method>POST</method>
        </list-item>
    </links>
</root>

POST

Description:

Using POST on this api to loan a book.

Request:

POST with book url in request body.

Name Description Type Required
book url of the book to loan. string True
Sample Request(body)
  • Form Data
  • XML
  • JSON
book http://waisvm-rfp1.ecs.soton.ac.uk/books/1 
<root>
    <book>http://waisvm-rfp1.ecs.soton.ac.uk/books/1</book>
</root>
                                        
{
    "book": "http://waisvm-rfp1.ecs.soton.ac.uk/books/1"
}
Response:
  • If accepted, return the detail of the loan and HATEOAS links.
  • If rejected, return the reason(reach the max number or have fine) and HATEOAS links.

  • Rejected

PATCH

Description:

Using PATCH on this api to renew or return a book.

Note:

The updated due date or status in PATCH parameter should be generated automatically by the website(i.e. click renew: due date postpone for 7 days, click return: status change to "Returned".)

Request:

Pass the loan id in the URL of PATCH call with information about the loan in request body.

Name Description Type Required
date_due Due date of book date False
status Status of the loan string False

  • Renew
  • Return
  • XML
  • JSON 
<root>
    <date_due>2015-11-22</date_due>
</root>

  • JSON 
<root>

PATCH

Description:

Using PATCH on this api to add or pay a fine.

Note:

The updated amount and status of fine in PATCH parameter should be generated automatically by the website(i.e. everyday when status is "Open": amount add 10, pay fine: status change to "Closed" and amount change to 0.)

Request:

Pass the fine id in the URL of PATCH call with information about the fine in request body.

Name Description Type Required
amount Fine amount Integer False
status Status of the fine string False
Sample Request(body)
  • Add Fine
  • Pay Fine
  • XML
  • JSON 
<root>
    <amount>20</amount>
</root>
                                        
{
    "amount": 20
}
  • XML
  • JSON 
<root>
    <status>Closed</status>
    <amount>0</amount>
</root>
                                        
{
    "status": "Closed",
    "amount": 0
}
