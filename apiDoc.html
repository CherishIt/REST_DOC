About

This is the REST API Document of ECS Library.

For each API, you can use HEAD to get the header of the response only.

For each API, you can use OPTIONS to get all supported HTTP Methods in the response header and information about this API in the response body(A sample response to "/authors" is shown on the right).

HTTP methods HEAD and OPTIONS WON'T be documented specifically in each API.

OPTIONS Response Sample

<?xml version="1.0" encoding="utf-8"?>
<root>
    <name>Author List</name>
    <description></description>
    <renders>
        <list-item>application/xml</list-item>
        <list-item>application/json</list-item>
        <list-item>text/csv</list-item>
    </renders>
    <parses>
        <list-item>application/xml</list-item>
        <list-item>application/json</list-item>
        <list-item>text/csv</list-item>
    </parses>
    <actions>
        <POST>
            <url>
                <type>field</type>
                <required>False</required>
                <read_only>True</read_only>
                <label>Url</label>
            </url>
            <name>
                <type>string</type>
                <required>True</required>
                <read_only>False</read_only>
                <label>Name</label>
                <max_length>255</max_length>
            </name>
            <owner>
                <type>field</type>
                <required>False</required>
                <read_only>True</read_only>
                <label>Owner</label>
            </owner>
        </POST>
    </actions>
</root>

Publishers

ECS library offers /publishers resource for providing the publishers list or creating a new publisher.

Once a new publisher is created, /publishers/<publisher_id> can be used to get, update or delete the publisher.

/publishers

Supported HTTP Methods: GET POST

GET

Description:

Using GET on this api to get a list of all publishers.

Request:

Simply GET /publishers

Sample Request
GET http://waisvm-rfp1.ecs.soton.ac.uk/publishers
Response:

Returns a list of all publishers.

Publisher:

Property Description
url Url of the publisher
name Name of the publisher
location Location of the publisher
owner Name of user who created the publisher
Sample Response
<?xml version="1.0" encoding="utf-8"?>
<root>
    <list-item>
        <url>http://waisvm-rfp1.ecs.soton.ac.uk/publishers/1</url>
        <name>O'Reilly Media</name>
        <location>Sebastopol, CA</location>
        <owner>rfp1n11</owner>
    </list-item>
    <list-item>
        <url>http://waisvm-rfp1.ecs.soton.ac.uk/publishers/2</url>
        <name>Visual Editions</name>
        <location>London, UK</location>
        <owner>rfp1n11</owner>
    </list-item>
    <list-item>
        <url>http://waisvm-rfp1.ecs.soton.ac.uk/publishers/3</url>
        <name>New Directions</name>
        <location>New York</location>
        <owner>rfp1n11</owner>
    </list-item>
</root>

POST

Description:

Using POST on this api to create a new publisher.

Request:

POST with information about new publisher in request body.

Name Description Required
name Name of the publisher True
location Location of the publisher False
Sample Request(body)
  • Form Data
  • XML
  • JSON
name publisher
location Southampton 
<root>
    <name>publisher</name>
    <location>Southampton</location>
</root>
                                        
{
    "name": "publisher",
    "location": "Southampton"
}
Response:

Returns the detail of the new publisher.

Publisher Properties
Sample Response
<?xml version="1.0" encoding="utf-8"?>
<root>
    <url>http://waisvm-rfp1.ecs.soton.ac.uk/publishers/193</url>
    <name>publisher</name>
    <location>Southampton</location>
    <owner>cz4n14</owner>
</root>

/publishers/<publisher_id>

Supported HTTP Methods: GET PUT PATCH DELETE

GET

Description:

Using GET on this api to get the detail of a publisher with the specific id.

Request:

Pass the publisher id in the URL of GET call.

Sample Request
GET http://waisvm-rfp1.ecs.soton.ac.uk/publishers/192
Response:

Returns detail of the publisher with the specific id.

Publisher Properties
Sample Response
<?xml version="1.0" encoding="utf-8"?>
<root>
    <url>http://waisvm-rfp1.ecs.soton.ac.uk/publishers/192</url>
    <name>publisher</name>
    <location>Southampton</location>
    <owner>cz4n14</owner>
</root>

PUT

Description:

Using PUT on this api to redefine a publisher's detail information.

Request:

Pass the publisher id in the URL of PUT call with all the information about the publisher in request body.

Name Description Required
name Name of the publisher True
location Location of the publisher True
Sample Request(body)
  • XML
  • JSON
<root>
    <name>publisher_redefined</name>
    <location>Southampton_redefined</location>
</root>
                                        
{
    "name": "publisher_redefined",
    "location": "Southampton_redefined"
}
Response:

Returns the detail of the updated publisher.

Publisher Properties
Sample Response
<?xml version="1.0" encoding="utf-8"?>
<root>
    <url>http://waisvm-rfp1.ecs.soton.ac.uk/publishers/193</url>
    <name>publisher_redefined</name>
    <location>Southampton_redefined</location>
    <owner>cz4n14</owner>
</root>

PATCH

Description:

Using PATCH on this api to modify part of a publisher's detail information.

Request:

Pass the publisher id in the URL of PATCH call with information about the publisher you want to modify in request body.

Name Description Required
name Name of the publisher False
location Location of the publisher False
Sample Request(body)
  • XML
  • JSON
<root>
    <name>publisher_redefined_patched</name>
</root>
                                        
{
    "name": "publisher_redefined_patched"
}
Response:

Returns the detail of the updated publisher.

Publisher Properties
Sample Response
<?xml version="1.0" encoding="utf-8"?>
<root>
    <url>http://waisvm-rfp1.ecs.soton.ac.uk/publishers/192</url>
    <name>publisher_redefined_patched</name>
    <location>Southampton_redefined</location>
    <owner>cz4n14</owner>
</root>

DELETE

Description:

Using DELETE on this api to delete a publisher with the specific id.

Request:

Pass the publisher id in the URL of DELETE call.

Sample Request
DELETE http://waisvm-rfp1.ecs.soton.ac.uk/publishers/192
Response:

Return 204 NO CONTENT to indicate this publisher no longer exists.

Books

ECS library offers /books resource for providing the books list or creating a new book.

Once a new book is created, /books/<book_id> can be used to get, update or delete the book.

/books

Supported HTTP Methods: GET POST

GET

Description:

Using GET on this api to get a list of all books.

Request:

Simply GET /books

Sample Request
GET http://waisvm-rfp1.ecs.soton.ac.uk/books
Response:

Returns a list of all books.

Book:

Property Description
url Url of the book
title Title of the book
pages Pages of the book
isbn13 Isbn-13 number of the book
date_published Publish date of the book
authors Authors of the book
publisher Publisher of the book
owner Name of user who created the book
